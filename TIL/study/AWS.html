# BASIC - SSH 로 Linux AWS Instance 연결

ssh -i [/path/key].pem [user-name]@[public-dns]
1

WARNING: UNPROTECTED PRIVATE KEY FILE! 🤯

.ssh/[my_private_key].pem 파일 권한에 대한 오류 이는 현재 프라이빗 키 파일이 권한 0777 으로 모든 사람에 대하여 이 파일에 읽기 및 쓰기가 허용되는 상태이다. 따라서 SSH 에서는 이 키를 사용 못하도록 하기 때문에

~ $ chmod 0400 .ssh/[my_private_key].pem

을 수행하여 권한을 변경해어 해결한다.

# BASIC - Elastic IPs 란?

  • Elastic IP 는 정적인 IPv4 주소로 Public IP 이다. (IPv6 주소는 지원 ❌)
  • Elastic IP 를 사용하면 정적 주소를 계정의 다른 인스턴스에 신속하게 다시 매핑하여 인스턴스나 소프트웨어의 오류를 마스킹할 수 있다.
  • 한 계정당 5개의 Elastic IP를 가질 수 있다.

Elastic IP 사용을 피해야 하는 경우

  • 랜덤한 public IP + DNS 👉 Load Balancer

# EC2 - Security Groups

Security Group 은 AWS 의 가장 fundamental 한 네트워크 보안으로 EC2 인스턴스에 대한 인바운드, 아웃바운드 룰을 관리하는 firewall 의 역할을 수행한다.

EC2에서 다음 내용을 기억하자 ❗️

  • 하나의 security group 은 여러개의 인스턴스에 사용될 수 있다.
  • EC2의 외부에 위치하기 때문에 트래픽이 블락되면 인스턴스까지 전달되지 않는다.
  • It's good to maintain one separate security group for SSH access
  • EC2 를 껏다키면 public IP 주소가 바뀐다 (바뀌지 않게 하기 위해선 elastic IP 사용)

Application 이 잘 동작하지 않는 경우

Case 1. 앱에 접근이 되지 않는 경우 (time out) 👉 Security Group 문제

Case 2. "connection refused" 👉 어플리케이션 상 에러

Security Group 을 지정해주지 않는 경우엔 default 가 사용된다.

Default Security Group

기본적인 디폴트 Security Group은 다음과 같다.

  • All inbound traffic (들어오는) is blocked by default
  • All outbound traffic is authorised by default

User Data

EC2 인스턴스를 시작할 때 인스턴스를 bootstrap (특정 명령어를 자동으로 수행) 할 수 있다. 이는 최초 실행 1회에 한하여 실행되며 root user 권한으로 실행된다.

Installing Updates, Installing Software, Downloading Common Files 등의 작업을 수행할 수 있다.

# EC2 - Instance Lauch Types (Pricing)⭐

# EC2 - On Demand

  • 쓰는만큼 지불, 가장 비싸며 running 상태인 시간 동안 만큼 비용 지불

Ex) 중단할 수 없는 불규칙한 단기 워크로드가 있는 애플리케이션의 경우

# EC2 - Reserved Instances

  • 1 ~ 3 년 사이 장기적인 목적이며 특정 인스턴스 타입을 reserve
  • On demand 에 비해 75% 절감

Ex) database

👉 Convertible Reserved Instance

  • 인스턴스 타입을 바꿀 수 있으나 54% 절감

👉 Scheduled Reserved Instances

  • 예약한 시간에만 동작하는 인스턴스
  • 매주 목요일 3 ~4시 에만 동작이 필요한 경우

# EC2 - Spot Instances

  • On demand 에 비해 90% 절감 가능
  • Ciritical job이나 데이터베이스에는 적합하지 않음

⭐️ Great Combo 👉 Reserved Instances for baseline + On-Demand & Spot for peaks

# EC2 - Dedicated Hosts

  • Full control of EC2 Instance placement
  • BYOL - Bring Your Own License (자사의 보유 라이선스를 활용)
  • 3 year period reservation
  • For companies that have strong regulatory (회사 규정 준수 요건을 충족)

대표적인 기능⭐️

  • 소켓 및 물리적 코어 수를 확인할 수 있다
  • 인스턴스 배치를 제어할 수 있음

# EC2 Instance Types

EC2 인스턴스는 다음과 같이 구성된다.

  • RAM (type, amount, generation)
  • CPU (type, make, frequency, generation, # of cores)
  • I/O (disk performance, EBS optimisations)
  • Network (network bandwitdth, network latency)
  • GPU
