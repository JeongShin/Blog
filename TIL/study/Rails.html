# Rails Study Notes

# Git Branch

처음 레일즈 Application 을 생성하면 디폴트로 master branch 에서 작업한다. 앞으로 Application 을 개발하면서 우리는 master branch 는 항상 deploy 가능한 상태를 유지해야 한다.

따라서 새로운 feature 를 추가한다고 가정하면 새로운 feature branch 를 생성하여 해당 branch 에서 작업을 이어가야 한다. 해당 feature branch 가 구현이 완료되고 테스트를 마치면 master branch 로 merge 된다.

# Branch 생성 예제

새로운 user model 을 추가하기 위해 아래와 같이 branch 를 생성해주었다.

branch 가 master 👉새로 생성한 branch 로 바뀐걸 볼 수 있다.

git checkout master

로 다시 master branch 로 돌아갈 수 있다.

# Branch Merge

새로운 feature 구현을 마쳤다면 다시 master branch 와 merge 해줘야 한다.

branch 에서 생성한 파일들이 master branch 로 정상적으로 merge 되어진 모습을 볼 수 있다.

# DB Association

# Many to Many Association

이번 클론 코딩 프로젝트의 데이터는 Articles Table 과 Categories Table 이 다대다 관계를 가진다.

즉, 하나의 article 은 여러개의 categories 를 가질 수 있고, 하나의 categories 는 여러개의 aritcles 을 가질 수 있다.

따라서 articles_categories 라는 연결 테이블을 사용하여 이들의 관계를 설정해 주었다.

이와 관련하여 다음과 같은 테스트를 구현하였다.

  1. Unit Tests : 모델에 적용되는 test로 단위별로 validation 등이 잘 적용되는지 테스트

  2. Functional Tests : 컨트롤러에 적용되는 test로, 예를들어 before_action 에 로그인 되지 않은 유저가 특정 불가능해야하는 action 을 수행하지는 않는지 등을 테스트

  3. Integration Tests : Full features, start to finish of a business process. Ex) a user signs up for the app
